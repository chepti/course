# שדרוג מערכת הקורס ל-v3 — הוראות התקנה

## מה השתנה

### תוסף Course Progress Tracker v3.0.0 (`course-progress-tracker.zip`)
נבנה מחדש ממבנה של קובץ אחד בן 4,067 שורות למבנה מודולרי:

```
course-progress-tracker.php   — קובץ ראשי קצר (טבלאות + טעינת מודולים)
includes/
  manifest.php   — 🆕 מבנה הקורס במקום אחד: יחידות, פרקים, דרישות השלמה
  progress.php   — חישוב התקדמות לפי המניפסט (עם תאימות לאחור)
  api.php        — 🆕 REST API (course/v1) עם פתרון אמיתי לבעיית הקאשינג
  ajax.php       — נקודות הקצה הישנות (נשמרו לתאימות לעמודים בקאש)
  shortcodes.php — דשבורד לומד חדש + כפתור "המשך מאיפה שעצרת"
  admin.php      — דוחות האדמין הקיימים (ללא שינוי התנהגות)
  enqueue.php    — טעינת הסקריפט
assets/course-tracker.js — סקריפט חדש: בלי jQuery, קריאה אחת במקום ארבע, retry אוטומטי
course-tracker.js        — הסקריפט הישן (נשאר לעמודים שעדיין בקאש)
```

**מה זה נותן:**
- **"המשך מאיפה שעצרת" אמיתי** — גם בתוך יחידה וגם בין יחידות, כולל מהדשבורד.
- **דשבורד לומד מלא** — מציג את כל יחידות הקורס כולל אלה שטרם התחילו (עד היום הוצגו רק יחידות עם פעילות).
- **שיקוף מדויק** — דרישות ההשלמה של כל פרק מוגדרות במניפסט (כמה סרטונים, תגובה, סימון ידני) במקום ניחושים לפי שם הפרק. סרטונים נספרים לפי מזהה ייחודי (לא ספירה כפולה).
- **יציבות** — nonce טרי לכל משתמש דרך admin-ajax (לא נשבר ע"י קאש), retry אוטומטי לשליחות שנכשלו, אבטחת CSRF הוחזרה.

### תוסף Course Registration v1.0.0 (`course-registration.zip`) — 🆕 תוסף נפרד
כל מנגנון הרישום הופרד לתוסף עצמאי, **בלי שינוי התנהגות**: רישום מ-Sumit, תפקיד "לומד בקורס", מייל ברוכים הבאים, אפילייאטים, אימות תשלומים, עמוד ההגדרות (Settings ‹ Course Registration), והשורטקוד `[course_thank_you]`. כל ההגדרות הקיימות נשמרות (הן יושבות ב-options של וורדפרס, לא בקבצי התוסף).

## סדר התקנה (חשוב!)

1. **גיבוי**: ודאי שיש גיבוי עדכני לאתר (או לפחות snapshot של ה-DB).
2. **התקיני קודם את Course Registration**: תוספים ‹ הוסף חדש ‹ העלאת תוסף ‹ `course-registration.zip` ‹ הפעלה.
   (שני התוספים יכולים לחיות יחד רגע — אין התנגשות שמות פונקציות עד שמעדכנים את הישן.)
3. **עדכני את Course Progress Tracker**: תוספים ‹ הוסף חדש ‹ העלאת תוסף ‹ `course-progress-tracker.zip` ‹ "החלף את הקיים" (Replace current with uploaded).
4. **בדיקה מהירה**: כניסה כתלמידה ליחידה ‹ לוודא שהניווט מסמן התקדמות ושמופיע כפתור "המשך מאיפה שעצרת".
5. **בניית המניפסט**: בתפריט האדמין ‹ Course Progress Tracker ‹ 🗺️ מבנה הקורס ‹ "בנייה אוטומטית מנתוני הפעילות" ‹ עריכת הכותרות והדרישות ‹ שמירה.
   - עד שהמניפסט מוגדר — הכל ממשיך לעבוד לפי הלוגיקה הישנה. המניפסט משדרג בהדרגה.

## מה לא צריך לעשות

- **אין צורך בקוד ב-functions.php** — התוסף טוען את הסקריפט בעצמו (עמודים תחת `aia` או יחידות במניפסט).
- **אין צורך לגעת בנתונים** — אותן טבלאות DB, כל ההתקדמות של 300+ הלומדים נשמרת.
- **אין צורך לשנות את ה-HTML של היחידות** — אותו חוזה `data-track-*` בדיוק.

## v3.1 - יחידות מתוך התוסף (שורטקוד במקום הדבקת HTML)

מגרסה 3.1.0 תוכן היחידות יושב בתוך התוסף (`units/unit-N.html`). **בעמוד וורדפרס שמים רק שורטקוד**:

| עמוד | מחליפים את כל בלוק ה-HTML ב: |
|---|---|
| יחידה 1 | `[course_unit id="1"]` |
| יחידה 2 | `[course_unit id="2"]` |
| ... | ... |
| יחידה 9 | `[course_unit id="9"]` |
| מפת יחידה 4 | `[course_unit id="4-map"]` |
| תרשים זרימה | `[course_unit id="flowchart"]` |

**מה זה נותן:** מעכשיו כל עדכון תוכן = העלאת זיפ אחד של התוסף (תוספים ‹ העלאה ‹ החלף את הקיים) — וכל היחידות מתעדכנות יחד. בלי להדביק HTML לעמודים יותר.

**איך מחליפים עמוד (פעם אחת, בלי סיכון):** עורכים את העמוד ‹ מוחקים את בלוק ה-HTML הגדול ‹ מוסיפים בלוק "שורטקוד" עם הקוד מהטבלה ‹ עדכון. ההתקדמות והתגובות לא מושפעות (הן שייכות לעמוד, לא לתוכן). אם משהו נראה לא טוב — מחזירים את בלוק ה-HTML מהגיבוי של גרסאות העמוד (Revisions).

מומלץ להחליף קודם עמוד אחד (למשל יחידה 1), לבדוק שהכל עובד — ניווט, סרטונים, מעקב, תגובות — ורק אז את השאר.

## שורטקודים

| שורטקוד | מה עושה |
|---|---|
| `[user_course_progress]` | דשבורד מלא (קיים — שודרג: כל היחידות + כפתור המשך) |
| `[course_resume_button]` | 🆕 רק כפתור "המשך מאיפה שעצרת" — לעמוד הבית של הקורס |
| `[unit_progress unit_id="123"]` | התקדמות יחידה בודדת (קיים; עכשיו גם בלי unit_id בתוך עמוד יחידה) |
| `[course_thank_you]` | עמוד תודה אחרי תשלום (עבר לתוסף Course Registration) |

## אם משהו משתבש

- שגיאה קריטית אחרי הפעלה → מחיקת תיקיית התוסף ב-FTP/מנהל קבצים והעלאת ה-zip הישן (`course-progress-tracker11.2.26.zip` שמור בריפו).
- הניווט לא מתעדכן → לנקות קאש (גם CDN/תוסף קאש) ולבדוק ב-Console שאין שגיאות.
- כל קבצי ה-PHP עברו `php -l` (בדיקת תחביר) לפני האריזה.
